Vision general del producto [Sistema de informacion]

  • Resumen de las caracteristicas del sistema

    Subsistema de gestión de ventas web

    El objetivo de este subsistema es capturar los datos del cliente y del pedido de los productos que realice para su posterior entrega, los pagos se realizarán con un subsistema de 3eros (Mercado Pago), pero se registrarán los datos de estos pagos para generar la factura, guardar los datos de las ventas y enviárselos a otro sistema de 3eros que es el sistema actual encargado a gestionar las ventas físicas del local

    Aclaracion

    El único medio de pago web sería gestionado por Mercado Pago, de manera que las 2 formas que este nos permite son por medio de una cuenta registrada en mercado pago y usar los medios de pago de esta o por pago único sin cuenta en el cual se piden los siguientes datos: Nombre, apellido, DNI, número de tarjeta de crédito/debido y su respectivo Código de seguridad CVV/CVC

    Descripcion del proceso

    Proceso de venta sin orden médica (online)

    El cliente visita la página web del comercio sin receta médica, selecciona el producto deseado o varios por medio de un carrito de compras y selecciona el local en el cual retirará los productos, procede al proceso de pago donde el sistema solicita la información necesaria y los datos del pago y el cliente los agrega, una vez realizado esto, se le genera la factura y se envía automáticamente al correo electrónico
