  • Casos de uso: Compra de productos web

    Actor principal: Cliente

    Personal involucrado e intereses:

    -Cliente: Quiere comprar productos de manera sencilla y rápida desde la web

    -Sistema de venta: Quiere recibir pedidos de compras, procesar las compras y registrarlas

    Precondiciones: El usuario debe haber ingresado a la página web

    Postcondición: El usuario realiza la compra de el/los productos, se registra la venta, se actualiza el stock, se genera la boleta y es enviada al usuario

    Escenario principal de éxito:

    1. El cliente ingresa a la página web
    2. El cliente selecciona el/los productos a comprar
    3. El cliente agrega el/los productos al carrito de compras
    4. El cliente selecciona a la sucursal a retirar los productos
    5. El sistema solicita al cliente el método de pago y lo respectivos datos para pagar
    6. El cliente carga los datos
    7. El cliente procede a realizar el pago de el/los productos
    8. El sistema registra la venta, envía la información de la venta al sistema externo de gestión de inventario
    9. El sistema genera el recibo, se lo muestra al cliente por pantalla y le envía una copia por medio de email
    10. El cliente cierra el sitio web

    Extenciones

    *a. En cualquier momento el Sistema falla:

    Para dar soporte a la recuperación y registro correcto, asegura que todos los estados y eventos significativos de una transacción puedan recuperarse desde cualquier paso del escenario.

    3. El sistema web se reinicia y retiene los productos que el cliente agregó al carrito.

    3a. El sistema detecta errores reteniendo los productos y pasa a un carrito limpio.

    4a. La sucursal no tiene los productos seleccionados por el usuario:

    1. El sistema muestra la opción de quitar los productos que no están en esa sucursal o de seleccionar otra sucursal

    7a. El cliente haya seleccionado pago con tarjeta y no fondos suficientes:

    1a. El cliente usa un método de pago alternativo

    1b. El cliente cancela la compra

Contratos

  • Contrato CO1: mostrarProductos

    Operación: mostrarProductos()

    Referencias Cruzadas: Caso de uso: Compra de productos web

    Precondiciones:

    -El cliente ha ingresado a la página web

    Post Condiciones:

    -Se devuelven los productos disponibles en la página web

  • Contrato CO2: seleccionSucursal

    Operación: seleccionSucursal()

    Referencias Cruzadas: Caso de uso: Compra de productos web

    Precondiciones:

    -El cliente ha ingresado a la página web.

    Post Condiciones:

    -Se devuelve a la sucursal seleccionada por el cliente.

  • Contrato CO3: seleccionarProducto

    Operación: seleccionarProducto(productoID, cantidad)

    Referencias Cruzadas: Caso de uso: Compra de productos web

    Precondiciones:

    -El cliente ha ingresado a la página web.

    Post Condiciones:

    -Se añade el producto y la cantidad seleccionada al carrito de compras del cliente.

  • Contrato CO4: procesoPago

    Operación: procesoPago(datosCliente, clienteID)

    Referencias Cruzadas: Caso de uso: Compra de productos web

    Precondiciones:

    -El cliente ha ingresado a la página web.

    Post Condiciones:

    -Se realiza el proceso de pago con los datos proporcionados por el cliente.

    -Devuelve [true] si el pago fue exitoso, [false] si no fue exitoso.

  • Contrato CO5: generarFactura

    Operación: generarFactura(clienteID)

    Referencias Cruzadas: Caso de uso: Compra de productos web

    Precondiciones:

    -Se ha realizado con éxito el proceso de pago.

    Post Condiciones:

    -Se genera una factura para la compra realizada y se asocia con el cliente.
